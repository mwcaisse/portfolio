Car Tracker

Web + Android Application

About the Project

Car Tracker is an Android application and web application to track vehicle and trip information. Similar to Strava and MapMyRun, except for your car. The Android application connects to your cars daignostic system using a bluetooth ODBII reader. Once connected it gathers various statistics, (speed, engine speed, oil temperature, air temperate, etc) and records them along with your current GPS position through out your trip. Once the trip is over they are uploaded to a server and available for viewing. Future plans are to update the server to provide more performance and maintance information. Such as how fuel efficient your car is and when its due for its next maintenance.

Related Skills

Java, Andriod SDK, HTML, Javascript, KnockoutJS, RequireJS, mybatis, MariaDB, SQL, Spring Framework, Jenkins

Project Approach

The project is divided into two applications. The Android Application which is responsible for collecting the information from the car and uploading it. The Web Application which collects the information from the Android app performs analysis and presents it to the user in an easy to read and understand format.

To keep things simgple and allow easy code re-use between the Android application and Web app both were written in Java.

The webapplication utilizes Spring Framework for hanlding dependency injection, the RESTFul API, serving WebPages using Thymeleaf as a templating engine, and to secure the application. MyBatis was used to provide a simple Data Access Layer between the application and its database.

On the android side piers odb-java-api is used to communicate with the vehicle. Spring RestTemplate is used to consume the RESTFul api exposed by the web application.
